# 18번째 테스트 (2026-07-13): outbox MarkProcessed 흡수 전/후 처리량 비교

## 커밋 해시

- **A-3 (흡수 전)**: `37a3e49` (feat(settlement): trade outbox 기반 write-ahead 정산)
- **opt (흡수 후)**: `bc8c00f` (perf(settlement): 정산 트랜잭션에서 outbox MarkProcessed 흡수해 DB 왕복 절반 축소)

## 왜 이 테스트를 했는지

[17번 벤치마크](17-2026-07-13-trade-outbox-throughput-impact.md)에서 A-3 outbox가
처리량을 ~17% 낮췄고, 원인이 정산 후 trade당 개별로 나가는 MarkProcessed UPDATE
44,300회임을 규명했다. 그 마킹을 정산 트랜잭션에 흡수(왕복 2회→1회)하는 최적화
(`bc8c00f`)가 실제로 처리량을 회복시키는지 검증한다.

## 왜 이 방식을 선택했는지 (방법론 — 세션 교란 배제)

처음엔 opt만 측정해 17번의 A-3(133.8 iter/s)·pre-A-3(162.7 iter/s)와 비교하려
했다. 그런데 opt가 **199.6 iter/s**로 나와, outbox 오버헤드가 있는데도 outbox가
아예 없던 pre-A-3를 넘어서는 물리적으로 이상한 결과였다.

원인은 **세션 교란**이었다. 17번은 VM stop 이전 세션에서, opt는 stop→start 이후
세션에서 측정됐는데, GCP가 재시작 시 다른 물리 호스트에 배정해 이번 세션이
전반적으로 빨랐다. 이를 확인·배제하기 위해 **A-3(`37a3e49`)를 이번 세션에서
재측정**해 opt와 같은 조건의 A/B를 만들었다. 두 빌드 모두 매 측정 전 빈
DB + backend 재시작, 동일한 hold 프로파일(1.5분 워밍업 → VU 3000에서 8분 유지).

## 실행한 정확한 커맨드

```bash
k6 run -e BASE_URL=http://10.10.0.3:8080 -e DEV_TOOLS_TOKEN=<토큰> ~/stress-hold3000.js
```

환경: 서버 `e2-medium`(2 vCPU), DB 별도 인스턴스, load-gen `e2-small`, k6 v2.0.0.

## 핵심 결과

### 같은 세션 A/B (세션 2, 순수 흡수 효과)

| 지표 | A-3(`37a3e49`) | opt(`bc8c00f`) | 변화 |
|---|---|---|---|
| 완료 iterations | 108,965 | 130,760 | **+20.0%** |
| 처리량 | 165.5 iter/s | 199.6 iter/s | **+20.6%** |
| `http_req_duration` p95 | 16.9s | 14.01s | **−17% (개선)** |
| med | 15.83s | 13.07s | −17% |
| `http_req_failed` | 0.00% | 0.00% | 동일 |

### 세션 교란의 증거 (동일 A-3 코드, 두 세션)

| 빌드 | 처리량 | 세션 |
|---|---|---|
| A-3 (`37a3e49`) | 133.8 iter/s | 세션 1 (17번, stop 전) |
| A-3 (`37a3e49`) | 165.5 iter/s | 세션 2 (18번, start 후) |

같은 코드가 세션 2에서 **+24% 빠르다** — VM 재시작으로 물리 조건이 달라졌다는
직접 증거다. opt가 pre-A-3를 넘어 보였던 것은 이 세션 효과였고, 같은 세션
A-3와 비교하면 순수 흡수 효과는 +20.6%다.

## 해석

1. **흡수는 A-3의 처리량 손실을 완전히 회복했다.** 같은 세션에서 A-3 대비
   +20.6%, p95 −17%. 17번이 규명한 원인(MarkProcessed 개별 왕복)을 정확히 제거한
   결과다.
2. **회복 수준은 pre-A-3와 동등하다.** 세션 보정 시(세션 2는 세션 1 대비 A-3
   기준 165.5/133.8 = 1.237배), pre-A-3를 세션 2로 환산하면 162.7×1.237 ≈ 201
   iter/s로 opt(199.6)와 거의 일치한다. 흡수 후 정산 경로의 DB 왕복이 pre-A-3와
   같으므로(트랜잭션 1개, outbox INSERT는 배치화돼 미미) 물리적으로도 타당하다.
   즉 **A-3의 정합성(크래시 유실 0)을 유지하면서 처리량 비용을 사실상 0으로
   되돌렸다.**
3. **세션 교란을 잡아낸 것 자체가 교훈이다.** opt만 단독 측정했다면 "흡수가
   pre-A-3보다 빠르다"는 틀린 결론을 남길 뻔했다. 클라우드 벤치마크는 세션 간
   물리 조건이 달라질 수 있으므로, A/B는 반드시 같은 세션에서 대조해야 한다.
4. **에러율은 세 측정 모두 0%.**

## 범위 밖 (Out of Scope)

- 시장가 완료(CompleteMarketOrder) 경로의 흡수 — 벤치마크(지정가 전용)에 없었고
  소량이라 이번 스코프 밖(설계 문서 참조). 시장가 비중이 큰 부하가 문제되면 후속.
- pre-A-3의 세션 2 재측정 — 세션 보정으로 충분히 설명되고, 핵심 A/B(A-3 vs opt)는
  같은 세션에서 확보했으므로 추가 측정은 하지 않았다.
